# 龙虾军团 (Lobster Army) - 多Agent协作系统

## 设计文档

日期：2026-03-29

---

## 一、系统概述

一个基于Claude Agent SDK的本地多Agent协作应用，提供Web可视化界面，支持多场景任务编排。用户提交需求后，协调者Agent分配任务给专业Agent，Agent间通过SDK内部通信协作完成任务，整个过程在Web界面实时可视化展示。

### 核心目标

- 将单个需求通过多Agent协作，从头脑风暴到验证提交完成全流程
- Agent间自主沟通协调，减少人工干预
- 可视化展示Agent协作过程，便于理解和追踪
- 支持多场景模板，可自定义Agent角色
- 打包为独立应用，供团队成员使用

---

## 二、系统架构

### 整体架构

```
本地应用 (单一进程)
    │
    ├── Web界面 (localhost:3000)
    │   ├── 任务管理
    │   ├── Agent状态可视化
    │   ├── 实时沟通面板
    │   ├── 角色配置
    │   └── 存储管理
    │
    ├── 协调者Agent (Claude Agent SDK)
    │   └── 任务分发、进度协调、冲突仲裁
    │
    ├── 专业Agent池 (最多5个活跃)
    │   └── 按场景模板配置的角色
    │
    └── 数据存储
        ├── SQLite (任务、日志)
        ├── JSON (配置)
        └── 文件 (产物)
```

### 单进程设计说明

所有Agent在同一Node.js进程内运行，Agent间通信由Claude Agent SDK内部管理，无需服务器中转。Web界面仅用于可视化和用户交互，通过WebSocket实时推送Agent状态更新。

## 三、Agent角色设计

### 协调者Agent

- 职责：接收用户需求、解析任务、分配给专业Agent、协调进度、汇总结果
- 能力：任务分解、进度跟踪、冲突调解、异常处理
- 位置：始终活跃，管理所有专业Agent

### 专业Agent池

最多5个Agent同时活跃，其他Agent处于待命状态，按需唤醒。每个Agent具备特定能力和工具权限。

## 五、角色自定义

用户可完全自定义每个Agent角色，包括：

- **角色名称**：自由命名
- **角色描述**：定义职责和行为模式
- **核心能力**：选择该角色具备的技能（文档理解、代码编写、测试执行等）
- **工具权限**：控制该角色能使用的工具（Read、Edit、Bash、WebFetch等）

首次使用时选择场景模板，之后可增删改角色配置，保存后下次启动自动加载。

## 六、任务执行流程

### 主流程

```
用户提交需求
     │
     ▼
协调者Agent解析需求，创建任务
     │
     ▼
根据场景模板分配给对应Agent
     │
     ▼
Agent协作循环（SDK内部通信）
     │
     ├─► 成功 → 汇总结果 → 完成
     │
     └─► 异常 → 协商解决 → 无法解决 → 人工介入
```

### Agent协作循环

1. 第一个Agent执行任务，输出结果和沟通信息
2. 协调者将结果传递给下一个Agent
3. Agent间可直接沟通协商（通过SDK SendMessage机制）
4. 所有沟通内容实时推送到Web界面显示
5. 完成后协调者汇总判断是否成功

### 异常处理

| 异常类型 | 处理方式 |
|---------|---------|
| 单个Agent执行失败 | 协调者分配给其他Agent重试 |
| Agent间意见冲突 | 协调者仲裁，或回退上一阶段重新规划 |
| 任务卡住无进展 | 通知用户，等待人工指令 |
| 环境/工具问题 | 运维Agent尝试修复，失败则人工介入 |

### 智能判断与人工干预

- Agent自主判断任务是否完成
- 异常时先尝试Agent间协商解决
- 协商无法解决时通知人工介入
- 用户可在任意时刻插入指令干预

## 八、存储设计

### 存储方式

| 存储内容 | 存储方式 | 说明 |
|---------|---------|------|
| 用户配置 | JSON文件 | 角色模板、自定义Agent配置 |
| 任务记录 | SQLite | 任务历史、执行状态 |
| 对话日志 | SQLite | Agent间完整对话内容 |
| 工作产物 | 本地文件夹 | Agent生成的代码、文档等 |

### 存储限制

| 资源类型 | 存储限制 | 说明 |
|---------|---------|------|
| 对话日志 | 最大1GB | 超过后提示清理，不自动删除 |
| 任务历史 | 最大1000条 | 超过后提示清理 |
| 工作产物 | 整体限制10GB | 包含在总存储中 |
| 总存储上限 | 10GB | 超过后弹窗提醒用户清理 |

### 内存管理

- 最多5个Agent同时活跃在内存中
- 其他Agent处于待命状态，按需唤醒
- 任务完成后释放不活跃Agent资源

### 清理功能

用户可在存储管理界面按需清理：
- 分类清理（对话日志、任务历史、工作产物）
- 按时间清理（30天前、已完成任务）
- 导出备份功能

## 十一、运行方式

### 开发阶段

直接运行Node.js，用户需安装Node环境：

```bash
npm install
npm run dev
# 打开浏览器访问 localhost:3000
```

成熟后可使用Electron打包为桌面应用，双击即用，无需安装Node环境。

1. 项目初始化 - 创建Next.js项目结构
2. Agent层开发 - 协调者Agent、场景模板定义
3. 后端API - 任务管理、WebSocket通信
4. 前端界面 - 任务管理、Agent可视化、沟通面板
5. 存储模块 - SQLite集成、文件管理、清理功能
6. 集成测试 - 完整流程验证
7. 打包发布 - 文档编写、使用说明

## 十四、总结

龙虾军团是一个本地运行的多Agent协作系统，通过Claude Agent SDK实现Agent间自主通信协作，Web界面提供可视化监控和用户干预入口。系统支持多种场景模板，角色可完全自定义，存储管理灵活可控。整体设计平衡了功能完整性和实现复杂度，适合作为团队效率工具使用。
